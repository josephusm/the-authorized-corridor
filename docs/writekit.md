## Quick start

```bash
wk check          # validate project
wk build html     # build as web page
wk build epub     # build as ePub ebook
wk build pdf      # build as PDF
wk build docx     # build as Word document
wk build md       # build as single Markdown
wk build all      # build all formats
wk watch          # watch for changes, rebuild automatically
wk stats          # word count, reading time, chapter balance
```

### Narrator vs POV

These are two distinct concepts:

- **`narrator`** — the narrative voice type. Set in `style.yaml` as the project default, overridable per chapter.
  - `first-person`: the narrator is a character ("I walked...")
  - `third-person`: external narrator following one character's thoughts ("He walked... he thought...")
  - `omniscient`: external narrator who knows everything ("Meanwhile, across town, she was thinking...")
  - `second-person`: rare ("You walk into the room...")

- **`pov`** — the point-of-view character(s). Who the reader experiences the scene through. Can be a single name or a list.
  - In first-person: who is the "I"
  - In third-person: whose thoughts we follow
  - In omniscient: optional, indicates the focus character(s) of the scene

## Print presets

| `royal` | 6.14×9.21in | full print |
| `lulu` | 6×9in + bleed | full print |

## Themes and fonts

```bash
wk theme list                  # available themes
wk theme use <name>            # switch theme
wk theme create <name>         # create custom theme
```

**Fonts**: place `.woff2`, `.ttf`, or `.otf` files in `assets/fonts/` or `themes/<name>/fonts/`. They are auto-embedded in HTML (base64) and ePub (ZIP entries). Reference them in your theme CSS.

**DOCX templates**: place a `template.docx` in `assets/` to customize Word output styles and fonts.
